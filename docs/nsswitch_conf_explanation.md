# /etc/nsswitch.conf 설명

이 문서는 리눅스에서 이름 해석과 계정 조회 순서를 정하는 `/etc/nsswitch.conf` 파일의 역할을 정리한 것이다. SSH 설정이나 방화벽처럼 눈에 보이는 서비스 설정과 달리, 이 파일은 시스템이 사용자, 그룹, 호스트 이름, 네트워크 서비스 정보를 어디서 먼저 찾을지 결정한다.

## 1. nsswitch.conf가 하는 일

`nsswitch`는 Name Service Switch의 약자다. 이름 그대로, 시스템이 필요한 정보를 조회할 때 어떤 데이터 소스를 어떤 순서로 사용할지 스위치처럼 정한다.

예를 들어 사용자를 조회할 때는 로컬 파일을 먼저 볼지, LDAP 같은 중앙 인증 서비스를 먼저 볼지, DNS를 먼저 볼지 같은 순서를 지정할 수 있다.

## 2. 자주 보는 항목

보통 다음 항목들이 많이 나온다.

- `passwd`: 사용자 계정 조회 순서
- `group`: 그룹 정보 조회 순서
- `shadow`: 암호 관련 정보 조회 순서
- `hosts`: 호스트 이름과 IP 주소 해석 순서
- `services`: 서비스 이름과 포트 매핑 조회 순서
- `networks`: 네트워크 이름 조회 순서
- `protocols`: 프로토콜 이름 조회 순서
- `ethers`: 이더넷 주소 조회 순서
- `rpc`: RPC 이름 조회 순서

## 3. 기본 형식

한 줄은 보통 다음처럼 구성된다.

`항목: 소스1 소스2 소스3`

예시:

`hosts: files dns`

이 뜻은 호스트 이름을 먼저 로컬 파일에서 찾고, 없으면 DNS를 조회하라는 의미다.

## 4. 자주 쓰는 소스

다음 소스가 가장 흔하다.

- `files`: `/etc/hosts`, `/etc/passwd` 같은 로컬 파일
- `dns`: DNS 서버
- `myhostname`: 현재 시스템의 호스트 이름
- `resolve`: systemd-resolved가 제공하는 해석 기능
- `systemd`: systemd가 제공하는 로컬 이름 조회 기능
- `nis`: NIS 같은 네트워크 정보 서비스
- `ldap`: LDAP 디렉터리 서비스

`systemd`는 아주 기본적인 공통 소스라기보다, systemd를 사용하는 배포판에서 특정 조회 항목과 함께 보이는 보조 소스에 가깝다.

## 5. 예시와 의미

### hosts

`hosts: files dns`

이 설정은 로컬 hosts 파일을 먼저 확인한 뒤, 실패하면 DNS를 사용한다.

### passwd

`passwd: files systemd`

이 설정은 로컬 사용자 계정을 먼저 보고, 그 다음 systemd 관련 사용자 정보를 확인한다.

여기서 `systemd`는 systemd가 관리하는 사용자 정보나 로컬 시스템 사용자 조회를 보완하는 역할을 한다.

### group

`group: files systemd`

그룹 정보도 같은 방식으로 조회한다.

즉, 로컬 파일에 없는 정보가 있으면 systemd 쪽 조회를 추가로 시도할 수 있다.

## 6. 왜 중요한가

`/etc/nsswitch.conf`는 다음 문제에 직접 영향을 준다.

- `ping`이나 `ssh`에서 호스트 이름이 IP로 바뀌는 방식
- `getent passwd` 같은 계정 조회 결과
- 로컬 파일과 중앙 인증 서비스 중 어디를 먼저 신뢰할지
- 시스템 기능이 예상과 다르게 동작하는 원인 파악

특히 `hosts` 순서가 잘못되면 이름 해석이 느려지거나, 로컬 설정이 무시되는 것처럼 보일 수 있다.

## 7. 확인 방법

파일 내용을 확인할 때는 다음을 사용한다.

`cat /etc/nsswitch.conf`

호스트 이름 해석이 제대로 되는지 확인할 때는 다음도 유용하다.

`getent hosts localhost`
`getent passwd root`
`getent group sudo`

`getent`는 nsswitch 설정을 실제로 따라가므로, 설정 확인에 적합하다.

## 8. 실무에서 기억할 점

- 순서는 중요하다. 앞에 있는 소스를 먼저 시도한다.
- `files`를 빼면 로컬 계정이나 hosts 파일이 기대대로 동작하지 않을 수 있다.
- 배포판과 서비스 구성에 따라 기본값이 조금씩 다를 수 있다.
- 잘못 수정하면 로그인, 이름 해석, 서비스 조회에 문제가 생길 수 있으니 주의해야 한다.

## 9. 한 줄 요약

`/etc/nsswitch.conf`는 리눅스가 사용자, 그룹, 호스트 이름 같은 정보를 어디서 어떤 순서로 찾을지 정하는 우선순위 설정 파일이다.
